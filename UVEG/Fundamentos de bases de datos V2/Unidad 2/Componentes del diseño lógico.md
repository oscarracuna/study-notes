
## [[Modelo entidad-relación]]
Consiste en una representación analítica de lo que se requiere almacenar en una base de datos y se encuentra compuesto por:
- Entidades
- Atributos
- Relaciones
- Reglas de negocio

**¿Qué es la entidad (gris) y el atributo (verde)?**
Entidad:
- La representación de un objeto, proceso o concepto del mundo real que se describe en una base de datos. Brinda información. Tiene una serie de características que definen su estado -> atributos.

Atributo:
- Son las características de las entidades, las definen y a su vez se almacenan en el modelado. Para que existan varios elementos de la misma entidad, los valores del atributo tienen que ser diferentes entre sí.

**Ejemplos:**

Entidad:
Un auto. Este tiene varias características, como la marca, color, precio, etc. <- atributos.

## ¿Qué son las relaciones?
Son el grado de asociación entre una o varias entidades que se pueden instanciar. Se pueden representar en forma de tablas para clarificar las reglas de negocio.
Tipos de relaciones:
- Relación uno a uno 1:1
- Relación de uno a muchos 1:n
- Relación de muchos a muchos n:m
![[Pasted image 20240623090842.png]]

## Reglas de negocio
Son las descripciones de las normas, políticas, definiciones y restricciones para el desarrollo de las actividades organizacionales que van de acuerdo con al visión del negocio. Utilizadas para la toma de decisiones del negocio.
