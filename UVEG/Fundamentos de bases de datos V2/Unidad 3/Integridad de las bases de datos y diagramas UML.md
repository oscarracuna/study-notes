
## Integridad de bases de datos
Es el reflejo de las reglas aplicadas a las relaciones de las tablas en una base de datos.

## [[Carga de datos]]
Para poder identificar la integridad de los datos es importante la carga de estos, ya que es la definición concreta y física de la base de datos que al ser llenada ayuda a corroborar que existe una correspondencia.

Para poder llenar o poblar la base de datos, ser requiere:
- Definición de vistas
- Tablas
- Dominios
- Esquemas

## Control de integridad
Se tienen que considerar varias aspectos, como lo son:
- Requerimiento de datos
	- Se requieren los datos que serán acumulados y las reglas de negocio que se tendrán
- Dominio de atributos
	- Se refiere a la implementación de los requerimientos tomados teniendo en cuenta los tipos de datos y la ejecución de las reglas de negocio.
- Integridad de identidades
	- Se requiere que ninguna de las tuplas en su clave principal sea nula. Esta sólo se usa en relaciones individuales o de tablas.
- Integridad referencial
	- Surge de las relaciones entre varias tablas y es la restricción entre dos de ellas.
- Restricciones generales
	- Son las reglas de negocio que no entran en ninguna de las definiciones anteriores.

## [[Diagramas UML]]
Unified Modeling Language.
Es un lenguaje usado para el desarrollo de proyectos de software. Son muy populares para casi todos los procesos y etapas en el desarrollo de proyectos de las tecnologías de la información.
