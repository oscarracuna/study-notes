
The front end interaction is mainly on the client-side so if it is attacked, it does not pose a direct threat to the core [[Back end]] of the web app and usually will not lead to permanent damage.

Although majority of the web app pen testing is focused on back end components and their functionality, it's important to also test front end components for potential vulnerabilities.

[[Sensitive Data Exposure]]
This refers to the availability of sensitive data in clear-text to the end user. Usually found in the [[Source Code]].
Sometimes we may find login credentials, hashes or other sensitive data hidden in the comments of a web page's source code or within external JS code being imported.
