
## Qué es un [[modelo de datos]]
Conjunto de herramientas conceptuales que describen las características de los datos, sus relaciones entre sí, así como sus restricciones.

## Modelos conceptuales
Son empleados para describir los datos de manera general. Presentan los datos de forma *similar* a cómo las personas entienden y visualizan el mundo real.
Ventajas:
- Capacidad de estructuración 
- Precisión de restricción de datos literales

## Modelos tradicionales
Fueron los primeros en usarse. Se utilizan para describir datos a nivel global y de manera lógica.
Los 3 más aceptados son:
- Modelo relacional
- Modelo de red
- Modelo jerárquico

## Modelos lógicos avanzados
Modelos más recientes sobretodo utilizadas en aplicaciones específicas que maneja nuevos y más complejos tipos de datos.

## Arquitectura de las [[bases de datos]]
Los modelos de datos manipulan la información contenida en los gestores.
Se dividen en tres niveles: 
- Externo
	-  El usuario final puede ver e interactuar con otros mediante las interfaces desarrolladas para su uso de una manera sencilla
- Conceptual
	- Nivel donde interactúan los datos con el código de las aplicaciones que utilizan los usuarios finales.
- Interno
	- Es el más cercano al almacenamiento físico de datos. Se encuentra totalmente alejado del usuario y sólo relaciona los datos con los requerimientos de la aplicación.
